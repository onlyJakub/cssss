  5. Jakie jest zastosowanie klas?

    Klasy w CSS (Cascading Style Sheets) mają kluczowe zastosowanie w stylizacji
    elementów na stronie internetowej. Pozwalają one na zdefiniowanie zestawu stylów,
    które mogą być następnie używane do różnych elementów HTML bez konieczności
    powtarzania tych samych reguł stylów.

  6. Kiedy ma zastosowanie lista klas?

    Lista klas w CSS ma zastosowanie w przypadkach, gdy chcemy przypisać
    elementowi więcej niż jedną klasę, aby stosować różne zestawy stylów lub
    dodatkowe dekoracje w zależności od potrzeb. Umożliwia to tworzenie bardziej
    elastycznych i reużywalnych reguł stylów.

  7. składnia teoretyczna listy klasy → obramowania, oraz uwagi.

    < selektor class="klasa1 klasa2 klasa3...">...
    Uwagi: • przy czym klasy: klasa1, klasa2, klasa3 muszą być zdeklarowane w wewnętrznym, zewnętrznym lub importowanym arkuszu stylów. • Kolejność wpisywania nazw klas w wartości atrybutu class="..." nie ma znaczenia na ostateczny sposób formatowania elementu. Jeżeli kilka przypisanych klas odnosi się do tej samej własności, o wyniku decyduje zasada kaskadowości.

  8. Czym są podzbiory klas. Składnia teoretyczna listy klasy → obramowania

    Istnieje możliwość określenia tzw. podzbioru klas, czyli podania, jakie klasy musi mieć jednocześnie przypisane element, aby wybrana reguła CSS miała do niego zastosowanie. Na przykład po wpisaniu:

    selektor.klasa1.klasa3 { cecha: wartość }
    zmieniony zostanie sposób formatowania elementu:
    < selektor class="klasa1 klasa2 klasa3...">...
    Kolejność podawania klas - zarówno w regule CSS, jak i w wartości atrybutu class="..." - nie ma znaczenia.

  9. Co jest sygnałem, że należy zastosować klasy?

    Zastosowanie klas w CSS jest korzystne, gdy kod zaczyna się powtarzać, co pomaga
    w unikaniu duplikacji i zwiększa elastyczność w modyfikacjach stylów. Klasy
    ułatwiają również modularyzację i organizację kodu w większych projektach, co
    przyczynia się do jego czytelności i zrozumiałości. Ponadto, klasy pozwalają na
    dynamiczne zmiany stylów w odpowiedzi na interakcje użytkownika i są użyteczne w
    responsywnym designie, dostosowując stylizację do różnych urządzeń.

  10. Kiedy i z jakiego powodu powinny być planowane klasy dla wykonywanej strony?

    Planowanie klas na wczesnym etapie projektowania strony pozwala na spójną i
    elastyczną stylizację, ułatwiając wielokrotne wykorzystanie zdefiniowanych
    zestawów stylów. Dzięki temu zmiany w stylach można wprowadzać konsekwentnie i
    efektywnie, minimalizując potrzebę modyfikacji kodu w różnych miejscach.
    Organizacja klasyfikacji na początku procesu projektowego wspiera czytelność i
    uporządkowanie kodu, co jest korzystne dla współpracy w zespołach i utrzymania
    spójności projektu.

  11. Jaki jest cel stosowania komentarzy podczas definiowania klas?

    Stosowanie komentarzy podczas definiowania klas w
    CSS pomaga w dokumentowaniu kodu, co ułatwia zrozumienie jego struktury i przeznaczenia przez
    innych programistów. Komentarze pozwalają też na szybkie odnalezienie i
    zrozumienie specyficznych sekcji kodu podczas przeglądania lub edycji stylów.
    Ponadto, dobrze umieszczone komentarze mogą służyć jako przypomnienie dla
    siebie samego w przyszłości, pomagając w późniejszych modyfikacjach i
    aktualizacjach stylów.

  13. Omów klasę donosząca się tylko do jednego selektora wraz z przykładem.

    Podobnie jak w przypadku identyfikatora, tak zdefiniowana klasa może zostać użyta tylko w
    tym selektorze, do którego została przypisana, np.:
    definicja arkusza styli
    p.moja
    {
    color:green;
    font-size:14pt;
    }
    wywołanie
    Powinna zostać wywołana w znaczniku < p> w następujący sposób:
    < p class=”moja”>Treść akapitu< /p>

  14. Jaki jest cel stosowania klasy uniwersalnej. Jakie jest znaczenie znaku „*” oraz kiedy można nie stosować tego znaku. Podaj przykład (definicja, wywołanie).

  1. W przypadku stosowanie klas, często podczas ich definiowania, nie wiemy jeszcze, w
    stosunku do jakich znaczników będziemy chcieli je później użyć. Wtedy przydatny okazuje
    się selektor uniwersalny, który pozwala przypisać własności klasy do wszystkich elementów,
    którym zostanie ona nadana, bez względu na typ znacznika czyli użycie jest możliwe w
    różnych znacznikach.
    Przykład:
    definicja arkusza styli
    *.przyklad_uniwersalna { color: red }
    Każdy element, któremu nadamy klasę przyklad_uniwersalna, będzie miał kolor czerwony.
    W tym przypadku gwiazdkę (*) w regule stylu można pominąć:
    .przyklad_uniwersalna { font-size:12 pt; color: red }
    Zwróć uwagę na fakt występowania kropki po selektorze lub przed nazwą klasy. Jest to
    element obowiązkowy i bez tego dalsza praca nie miałaby większego sensu.
    Jak widać na powyższym przykładzie, klasa jest wywoływana w dokumencie za pomocą
    atrybutu class, który może zostać umieszczony niemal w dowolnym znaczniku kodu HTML.
    Przykład
    definicja arkusza styli
    .moja
    {
    color:red;
    font-size:12pt;
    font-style:italic;
    }
    wywołanie
    Wywołanie klasy tak określonej klasy również odbywa się za pomocą atrybutu class.
    < p class=”moja”>Przykład akapitu z określoną klasą
    < ol class=”moja”>
    < h1 class=”moja”>Przykład nagłówka z określoną klasą
    Przykład
    Listing arkusza CSS
    p
    {
    font-size:14pt;
    color:green;
    }
    .moja
    {
    font-style:italic;
    font-size:20pt;
    color:red;
    }
    Listing fragmentu kodu XHTML/HTML
    < p>Blok tekstu o kolorze zielonym i czcionce 14pt
    < p class="moja">Blok tekstu, w którym użyłem klasy o nazwie .moja
    < p>Blok tekstu o kolorze zielonym i czcionce 14pt
    < ol class="moja">
    < li>lista, w której użyłem klasy o nazwie .moja
    < li>lista, w której użyłem klasy o nazwie .moja
